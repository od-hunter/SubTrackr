# Internationalization (i18n)

SubTrackr uses `i18next` and `react-i18next` for multilingual support, including Right-to-Left (RTL) layout handling.

## Structure

```text
src/i18n/
  ├── locales/
  │     ├── en.json    # English
  │     ├── hi.json    # Hindi
  │     └── ar.json    # Arabic (RTL)
  └── config.ts        # i18next initialization
```

## Usage

### In Functional Components
Use the `useTranslation` hook:
```tsx
import { useTranslation } from 'react-i18next';

const MyComponent = () => {
  const { t } = useTranslation();
  return <Text>{t('common.save')}</Text>;
};
```

### Format Locale-Aware Data
Use the `languageService` for dates and currency:
```tsx
import { languageService } from '../services/i18n';

// Results vary by active language (€10.00, ₹10.00, etc.)
const price = languageService.formatCurrency(10, 'USD'); 

// Format dates (DD/MM/YYYY vs MM/DD/YYYY)
const date = languageService.formatDate(new Date());
```

## Adding a New Language

1. Create a new JSON file in `src/i18n/locales/` (e.g., `fr.json`).
2. Import and add it to `RESOURCES` in `src/i18n/config.ts`.
3. Add the language to the `LANGUAGES` array in `LanguageSettingsScreen.tsx`.

## RTL Support

- When the language is set to Arabic (`ar`), `I18nManager.forceRTL(true)` is called.
- React Native layout automatically flips flex directions and margins.
- **Tip**: Use `paddingStart`/`paddingEnd` instead of `paddingLeft`/`paddingRight` for automatic RTL adjustment.

## Persistence

The selected language is saved in `AsyncStorage` under the key `@subtrackr_language` and re-applied during app launch in `initI18n()`.
